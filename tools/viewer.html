How to use this tool

Loading data

  • Auto-load both works if you served this folder via a local web server (e.g. python -m http.server 8000 from the trade space explorer root, then open http://localhost:8000/tools/viewer.html).
  • Otherwise click Load technologies.jsonl and pick data/technologies.jsonl; then click Load subsystems.json and pick data/subsystems.json.

System Designer workflow

  1. Look at the subsystem grid. Each card is a major subsystem you need to fill.
  2. For each card, the candidate techs are filtered from the DB by the subsystem's match_categories, match_tags, and expected_outputs rules in subsystems.json. Edit that file to refine matching.
  3. Click any candidate tech to add it to the subsystem. Click again to remove. Multiple techs per subsystem = parallel/redundant operation.
  4. Watch the Integration view live-update. Green rows = your architecture closes that flow. Red = deficit. Yellow = surplus (vent or use). Blue = orphan / bridge slot.
  5. For each blue/red row, drop into Bridge slots, label what tech is needed, and assign it as a research action to the team.

Editing subsystems.json

The subsystem decomposition isn't fixed — it's the team's working model. Edit data/subsystems.json to add, remove, or refine subsystems. Each subsystem entry needs:

  • id — unique slug
  • name, description
  • match_categories — array of tech-category strings to filter candidates
  • match_tags (optional) — array of tag strings to also include
  • expected_outputs / expected_inputs — flow_ids the team expects this subsystem to handle
  • allow_parallel — true if multiple techs can fill it
  • owner_subteam — ST1, ST2, ST3, ST4
